## 低危服务

- `192.168.50.201:22` - SSH / OpenSSH 9.6p1 Ubuntu 3ubuntu13.11  
  - **风险**：SSH 通常安全性较高，但如允许弱口令登录、默认口令或不安全的身份认证方式，或发生版本漏洞，可能导致未授权访问。此外，暴露在外网会被持续爆破尝试。  
  - **攻击方式**：
    - 暴力破解（爆破）密码或用户名
    - 社会工程钓鱼窃取凭据
    - 利用已知 SSH 用户名和弱口令
    - 针对特定版本的潜在零日利用
  - **公开漏洞**：
    - 截至目前 OpenSSH 9.6p1 没有被公开报道有致命远程代码执行（RCE）或权限提升的高危漏洞。
    - 近期相关 CVE：
      - [CVE-2023-51385](https://nvd.nist.gov/vuln/detail/CVE-2023-51385)：针对 OpenSSH 代理转发的 DoS（影响度低，非默认开启且环境受限）。
      - [CVE-2024-6409](https://nvd.nist.gov/vuln/detail/CVE-2024-6409)：缓冲区溢出，受影响的版本主要为 8.5p1 及更早版本，9.x 不受影响。
    - 常见爆破字典攻击可利用 Hydra、Medusa、ncrack 等工具。
    - 当前版本没有已知 Exploit DB 高危利用。

---

**总结**：  
此主机暴露了 SSH 服务，建议关闭外部不必要访问、使用强认证策略（强密码或密钥登录）、限制登录来源 IP，并持续关注 OpenSSH 官方安全公告。当前版本无严重公开漏洞，但应预防暴力破解风险。
